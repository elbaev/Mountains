# Project instructions

## Critical content rules
- Do not compress or simplify the long-form Russian copy unless explicitly asked.
- Preserve full explanatory text blocks; summaries may appear only as an additional UI layer.
- Do not mention the guests' age anywhere in UI copy.
- The interface must remain elegant, calm, premium, and highly readable.
- Build the app as a content-first travel guide, not as a promo landing page.

## Product goals
- Browser web app
- React
- Stylish navigation
- Sections, tabs, accordions, cards, checklist
- Full Russian text preserved from the spec

## Implementation rules
- Keep long text in dedicated data fields.
- Do not replace meaningful paragraphs with short bullet fragments.
- Prioritize readability on desktop and mobile.
- The interface should help the user navigate the guide, not reduce the guide to decorative fragments.

## Content architecture
- Each major section should have both a short summary layer and a full long-text layer.
- Long-form content must remain accessible on every key screen.
- Route plans must keep explanatory prose, not just lists of places.
- Hotel descriptions must remain informative, with positioning by scenario and comfort level.

## UI direction
- Premium minimalism
- Calm mountain editorial style
- Strong typography
- Spacious layout
- Clear navigation
- Good mobile readability
